- [Minimal Example](https://stackblitz.com/github/trpc/trpc/tree/main/examples/minimal?file=server%2Findex.ts&file=client%2Findex.ts&view=editor) - 一个最简的 Node.js http 服务器和一个调用服务器上的函数并将请求输出到控制台的客户端。

## 使用 tRPC

"使用 tRPC" 对不同的人意味着不同的事情。本页面的目标是根据你的目标为你提供正确的资源指南。

### 在现有的 tRPC 项目中提高生产力

- 阅读[概念](./concepts.mdx)页面。
- 熟悉[路由](../server/routers.md)、[过程](../server/procedures.md)、[上下文](../server/context.md)和[中间件](../server/middlewares.md)。
- 如果你使用 React，请阅读有关 [useQuery](../client/react/useQuery.md)、[useMutation](../client/react/useMutation.md) 和 [useContext](../client/react/useContext.mdx)的文档。

### 创建一个新项目

由于 tRPC 可以存在于许多不同的框架中，你首先需要决定你想在哪里使用它。

在后端，有适用于多个框架以及原生 Node.js 的[适配器](../server/adapters-intro.md)。在前端，你可以使用我们的 [React](../client/react/introduction.mdx) 或 [Next.js](../client/nextjs/introduction.mdx) 集成，也可以使用各种其他框架的[第三方集成](../community/awesome-trpc.mdx#frontend-frameworks)，或者工作在任何支持 JavaScript 的地方的[原生客户端](../client/vanilla/setup.mdx)。

选择了你的技术栈之后，你可以使用[模板](./example-apps.mdx)来构建你的应用程序，或者根据所选的后端适配器和前端集成的文档从头开始构建。

### Adding tRPC to an existing project
### 将 tRPC 添加到现有项目

将 tRPC 添加到现有项目与创建新项目并没有太大差异，因此资源都是通用的。主要的挑战在于如何将 tRPC 与现有应用程序集成。以下是一些建议：

- 你不需要将所有现有的后端逻辑迁移到 tRPC。常见的迁移策略是最初仅在新端点中使用 tRPC，然后再逐步将现有端点迁移到 tRPC。
- 如果不确定从何处开始，请查阅你选择的后端[适配器](../server/adapters-intro.md)和前端实现的文档，以及[示例应用](./example-apps.mdx)。
- 如果你正在寻找关于如何在较大代码库中使用 tRPC 的灵感，可以在 [使用 tRPC 的开源项目](../community/awesome-trpc.mdx#-open-source-projects-using-trpc) 中找到一些示例。
