## 简介

<p>
  <abbr title="TypeScript Remote Procedure Call">tRPC</abbr> 允许你轻松创建和使用完全符合类型安全的 API，无需通过 schemas 或代码生成的方式来实现。
</p>

随着 TypeScript 和静态类型在 Web 开发中逐渐成为最佳实践，API 接口规范成为一个主要的痛点。我们需要更好的方法来为我们的 API 接口 **提供静态类型**，并在客户端和服务器（或服务器之间）之间 **共享这些类型**。我们充分利用了现代 TypeScript 的全部功能着手构建了一个简单的库，用于构建类型安全的 API。

### 传统 REST 或 GraphQL 的一个替代方案

目前，GraphQL 是在 TypeScript 中实现类型安全 API 的主流方式（[而且它非常棒！](../further/further-reading.md#relationship-to-graphql)）。由于 GraphQL 是为实现 API 而设计的与语言无关的规范，它无法充分发挥像 TypeScript 这样的语言的强大功能。

如果您的项目全栈都是使用 TypeScript 构建的，您可以**直接**在客户端和服务器之间共享类型，而无需依赖代码生成。

### 谁是 tRPC 的目标对象

tRPC 适用于全栈都使用 TypeScript 的开发人员。它可以轻松地编写端点，然后你可以安全地在应用程序的前端和后端中使用这些端点。在构建时，它可以捕获与 API 规范相关的类型错误，从而减少运行时在应用程序中的错误。

## 特点

- ✅&nbsp; 经过充分的测试，适用于生产环境。
- 🧙‍♂️&nbsp; 客户端具有完整的静态类型安全性和自动补全功能，包括输入、输出和错误。
- 🐎&nbsp; 优秀的开发体验 - 无需代码生成、运行时膨胀或构建流程。
- 🍃&nbsp; 轻量级 - tRPC 没有任何依赖，客户端占用空间很小。
- 🐻&nbsp; 适用于新项目和旧项目 - 它易于入门，也可以添加到现有的老项目中。
- 🔋&nbsp; 框架无关 - tRPC 社区为所有最流行的框架构建了[适配器](https://trpc.io/docs/awesome-trpc#-extensions--community-add-ons)。
- 🥃&nbsp; 支持订阅 - 为你的应用程序添加类型安全的观测。
- ⚡️&nbsp; 请求批处理 - 同时发出的请求可以自动合并为一个请求。
- 👀&nbsp; 示例 - 查看[示例](example-apps.mdx)以学习或作为项目的起点。
